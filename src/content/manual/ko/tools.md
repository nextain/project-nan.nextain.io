도구가 활성화되면 AI 아바타가 다양한 작업을 직접 수행할 수 있습니다.

## 사용 가능한 도구

| 도구 | 기능 | 보안 |
|------|------|------|
| execute_command | 터미널 명령 실행 | T3 |
| read_file | 파일 내용 읽기 | T0 |
| write_file | 파일 생성/수정 | T2 |
| search_files | 파일 시스템 검색 | T0 |
| web_search | 인터넷 검색 | T1 |
| apply_diff | 파일 편집 (diff 적용) | T2 |
| browser | 웹 페이지 열기 | T1 |
| sessions_spawn | 서브 에이전트 생성 | T2 |

## 사용 예시

대화에서 자연어로 도구를 사용할 수 있습니다:

| 요청 | 사용되는 도구 |
|------|-------------|
| "이 폴더에 있는 파일 보여줘" | search_files |
| "README.md 읽어줘" | read_file |
| "git status 실행해" | execute_command |
| "TypeScript에 대해 검색해줘" | web_search |
| "이 코드 리뷰해줘" | sessions_spawn |

## 보안 등급 및 승인 정책

AI가 도구를 실행할 때 발생할 수 있는 잠재적 위험을 방지하기 위해 4단계 보안 등급을 적용합니다. 각 등급에 따라 사용자의 개입 여부가 결정됩니다.

| 등급 | 명칭 | 설명 | 승인 방식 |
|:---:|:---:|---|:---:|
| **T0** | **읽기 전용** | 파일 읽기, 날짜 확인 등 시스템에 변경을 주지 않는 조회 작업 | **자동 승인** |
| **T1** | **알림** | 인터넷 검색 등 외부 서비스에 접근하지만 로컬 데이터는 수정하지 않음 | **알림만 표시** |
| **T2** | **주의 필요** | 파일 수정, 새로운 에이전트 생성 등 시스템 상태를 변경하는 작업 | **사용자 승인 필요** |
| **T3** | **위험** | 터미널 명령 실행 등 시스템에 직접적인 영향을 줄 수 있는 위험한 작업 | **매회 명시적 승인** |

처음 특정 도구 승인 시 "항상 허용"을 선택하면, 이후 동일한 도구는 사용자 확인 없이 자동으로 실행됩니다. 보안 설정은 **설정 > 도구** 탭에서 언제든지 초기화할 수 있습니다.

## Gateway 연결

도구를 사용하려면 Gateway가 연결되어야 합니다:

1. 설정 > 도구에서 **도구 활성화**를 켭니다
2. 일반 사용자 환경에서는 Gateway 연결 정보가 자동 처리되므로 추가 입력 없이 바로 사용합니다
3. 도구가 반응하지 않으면 앱을 재시작한 뒤 다시 시도합니다
4. 스킬 탭에서 필요한 스킬이 활성화되어 있는지 확인합니다
5. 자체 Gateway를 운영하는 경우에만 Gateway URL/토큰을 직접 확인합니다
