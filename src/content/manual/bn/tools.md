## উপলব্ধ টুলস

| Tool | Function | Security |
|------|----------|----------|
| execute_command | টার্মিনাল কমান্ড চালান | T3 |
| read_file | ফাইলের বিষয়বস্তু পড়ুন | T0 |
| write_file | ফাইল তৈরি/পরিবর্তন করুন | T2 |
| search_files | ফাইল সিস্টেম অনুসন্ধান করুন | T0 |
| web_search | ইন্টারনেট অনুসন্ধান করুন | T1 |
| apply_diff | ফাইল এডিট করুন (ডিফ প্রয়োগ করুন) | T2 |
| browser | ওয়েব পেজ খুলুন | T1 |
| sessions_spawn | সাব-এজেন্ট তৈরি করুন | T2 |

## ব্যবহারের উদাহরণ

আপনি স্বাভাবিক ভাষার মাধ্যমে টুলস ব্যবহার করতে পারেন:

| Request | Tool Used |
|---------|-----------|
| "এই ফোল্ডারে ফাইলগুলি দেখান" | search_files |
| "README.md পড়ুন" | read_file |
| "git status চালান" | execute_command |
| "TypeScript অনুসন্ধান করুন" | web_search |
| "এই কোডটি পর্যালোচনা করুন" | sessions_spawn |

## নিরাপত্তা অনুমোদন

প্রতিটি টুলের জন্য তার নিরাপত্তা স্তরের উপর ভিত্তি করে অনুমোদনের প্রয়োজন:

- **T0 (Read-only)**: স্বয়ংক্রিয়ভাবে অনুমোদিত — কোনো পার্শ্বপ্রতিক্রিয়া নেই
- **T1 (Notification)**: বিজ্ঞপ্তি দেখানো হবে — বাহ্যিক অ্যাক্সেস কিন্তু কোনো পরিবর্তন নেই
- **T2 (Caution)**: ব্যবহারকারীর অনুমোদন প্রয়োজন — ফাইল পরিবর্তন, এজেন্ট তৈরি
- **T3 (Dangerous)**: ব্যবহারকারীর অনুমোদন প্রয়োজন — কমান্ড কার্যকর করা

প্রথমবার অনুমোদন করার সময়, "Always Allow" নির্বাচন করলে সেই টুলটি স্বয়ংক্রিয়ভাবে ভবিষ্যতে ব্যবহারের জন্য অনুমোদিত হবে। আপনি Settings > Tools-এ অনুমোদিত টুলস রিসেট করতে পারেন।

## গেটওয়ে সংযোগ

টুলস ব্যবহারের জন্য একটি গেটওয়ে সংযোগের প্রয়োজন:

1. Settings > Tools-এ **Enable Tools** সক্ষম করুন
2. সাধারণ ব্যবহারকারীর পরিবেশে, গেটওয়ে সংযোগের বিবরণ স্বয়ংক্রিয়ভাবে পরিচালিত হয়
3. যদি টুলস সাড়া না দেয়, অ্যাপটি পুনরায় চালু করুন এবং আবার চেষ্টা করুন
4. Skills ট্যাবে প্রয়োজনীয় দক্ষতাগুলি সক্ষম করা আছে কিনা তা পরীক্ষা করুন
5. শুধুমাত্র স্ব-হোস্টেড/উন্নত সেটআপের জন্য ম্যানুয়াল Gateway URL/token যাচাইকরণ প্রয়োজন
