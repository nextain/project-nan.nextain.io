## أنواع المهارات

### المهارات المدمجة
مدمجة في التطبيق — لا يمكن تعطيلها:

| المهارة | الوظيفة | مستوى الأمان |
|-------|----------|---------------|
| `skill_time` | التحقق من التاريخ/الوقت الحالي | T0 |
| `skill_memo` | حفظ/استرجاع المذكرات | T0 |
| `skill_system_status` | التحقق من حالة النظام | T0 |
| `skill_weather` | التحقق من الطقس | T0 |
| `skill_notify_slack` | إرسال الإشعارات عبر Slack webhook | T1 |
| `skill_notify_discord` | إرسال الإشعارات عبر Discord webhook | T1 |
| `skill_skill_manager` | إدارة المهارات: البحث، التمكين، التعطيل | T0 |

### المهارات المخصصة
تُضاف عبر Gateway — يمكن تشغيلها/إيقافها:
- قراءة/كتابة الملفات، تنفيذ الأوامر، البحث في الويب، إلخ.
- نوع Gateway أو Command

## مصادر المهارات (من أين تأتي؟)

- **المهارات المدمجة**: مجمعة مع التطبيق
- **المهارات المخصصة**: تُحمّل من بيانات المهارات المحلية (على سبيل المثال، `~/.naia/skills/.../skill.json`)
- وسّع بطاقة المهارة للتحقق من شارة `source` الخاصة بها

## كيفية إضافة مهارة مخصصة

يتوافق Naia OS بنسبة 100% مع نظام OpenClaw البيئي. هناك ثلاث طرق لإضافة المهارات:

### 1. اطلب من الذكاء الاصطناعي بناءها (الأسهل)
اشرح ما تريده في الدردشة، وسيقوم AI 아바타 بكتابة الكود وإنشاء المهارة لك.
> "ابنِ مهارة تجلب سعر الصرف الحالي وتحفظه في `~/.naia/skills/exchange/skill.json`."

### 2. التثبيت من Clawhub (طريقة OpenClaw)
يمكنك استخدام أداة Terminal (`execute_command`) لتثبيت المكونات الإضافية مباشرة من **[Clawhub.ai](https://clawhub.ai)**، وهو سجل مهارات OpenClaw الرسمي الذي يحتوي على أكثر من 5,700 مهارة.

> "شغّل `openclaw plugins install @openclaw/plugin-github` في الطرفية لتثبيت المكون الإضافي Github."

⚠️ **تحذير أمني:** المهارات التي يتم تنزيلها من Clawhub أو مصادر أخرى عبر الإنترنت يكتبها أطراف ثالثة. قبل التثبيت، اطلب دائمًا من الذكاء الاصطناعي **"مراجعة كود هذه المهارة بحثًا عن أي مخاطر أمنية (مثل حذف الملفات أو سرقة المعلومات الشخصية) قبل المتابعة."**

### 3. الإضافة اليدوية
1. أنشئ بيان مهارة في `~/.naia/skills/<skill-name>/skill.json`
2. ضع أي نص برمجي/قابل للتنفيذ مطلوب لتلك المهارة في نفس المجلد
3. افتح علامة تبويب المهارات وتحقق مما إذا كانت المهارة الجديدة تظهر
4. قم بتمكينها باستخدام زر التبديل
5. اختبرها من الدردشة بطلب من شأنه تشغيل المهارة

إذا لم تظهر، أعد تشغيل التطبيق وتحقق مرة أخرى.

## تكامل مجتمع Botmadang

يتضمن Naia OS مهارة مدمجة (`skill_botmadang`) مخصصة لـ **Botmadang**، وهو مجتمع وكلاء الذكاء الاصطناعي الكوري.

يمكنك توجيه AI 아바타 لبدء أنشطته على Botmadang عبر الدردشة:
> "سجّل كوكيل جديد في Botmadang. عيّن اسمك إلى 'naia Agent'."

بمجرد التسجيل وتوفير API Key، يمكن لـ AI 아바타 نشر المقالات أو التعليق على سلاسل رسائل الوكلاء الآخرين بشكل مستقل.

## مهارات الإشعارات (Slack / Discord / Google Chat)

تعد `skill_notify_slack` و `skill_notify_discord` مهارات إشعارات مدمجة ترسل الرسائل عبر webhooks.

### إعداد Webhook

لاستخدام مهارات الإشعارات، تحتاج إلى تكوين عنوان URL لـ webhook. هناك طريقتان:

**الطريقة 1: متغيرات البيئة (موصى بها)**

```bash
# Add to ~/.bashrc or ~/.zshrc
export SLACK_WEBHOOK_URL="https://hooks.slack.com/services/T.../B.../xxx"
export DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/123/abc"
```

**الطريقة 2: config.json**

```json
// ~/.naia/config.json
{
  "notifications": {
    "slack": {
      "webhookUrl": "https://hooks.slack.com/services/T.../B.../xxx"
    },
    "discord": {
      "webhookUrl": "https://discord.com/api/webhooks/123/abc"
    }
  }
}
```

> تأخذ متغيرات البيئة الأولوية على config.json.

### أمثلة الاستخدام

ما عليك سوى سؤال AI 아바타 في الدردشة:

- "أرسل إشعار 'deploy complete' إلى Slack"
- "انشر تقرير حالة الخادم إلى Discord"
- "أبلغ قناة #ops بنتائج البناء"

سيقوم AI 아바타 تلقائيًا باستدعاء `skill_notify_slack` أو `skill_notify_discord`.

إذا لم يتم تكوين أي webhook، فسيتم عرض رسالة تشرح خطوات الإعداد.

### تكامل OpenClaw Gateway (متقدم)

عند توصيل OpenClaw Gateway، ستحاول مهارات الإشعارات أولاً استخدام `skills.invoke` RPC الخاص بـ Gateway. إذا فشل ترحيل Gateway، تعود المهارة إلى التسليم المباشر عبر webhook.

يوفر تكامل قناة Gateway ميزات أغنى (تنسيق الرسائل، سلاسل الرسائل، الإشارات، إلخ).

## سيناريو متقدم: OpenClaw + أتمتة cron

في إعدادات الأتمتة الجماعية/الشخصية، يمكنك تسجيل المهارات في OpenClaw وتشغيلها بجدول زمني باستخدام cron.

سيناريوهات الأمثلة:
- يوميًا 09:00: إنشاء ملخص لسجلات عمل الأمس
- كل ساعة: مسح مجلد مستهدف والإبلاغ عن أي حالات شاذة
- منتصف الليل: إنشاء ورفع تقرير يومي

السير الموصى به:
1. سجّل المهارة المخصصة وتحقق منها محليًا أولاً
2. قم بتكوين webhooks مهارات الإشعارات لربط قنوات التنبيه
3. أضف خطوة استدعاء مهارة في تعريف مهمة OpenClaw الخاصة بك
4. أرفق جدول cron كمشغل متكرر
5. أضف سياسات إعادة المحاولة/الإشعارات في حالة الفشل

> **خريطة الطريق**: ستتوفر واجهة مستخدم لجدولة cron، ودعم Telegram، وتوجيه متعدد القنوات (إرسال رسالة واحدة إلى قنوات متعددة في وقت واحد) في التحديثات المستقبلية.

## بطاقات المهارات

تُعرض كل مهارة كبطاقة:

![تفاصيل بطاقة المهارة](skills-card.png)

- **الاسم**: اسم المهارة (على سبيل المثال، `skill_read_file`)
- **الوصف**: ملخص من سطر واحد (قد يكون مبتورًا)
- **النقر**: انقر على البطاقة لتوسيع الوصف الكامل
- **الشارات**: النوع (مدمج/gateway/command)، مستوى الأمان (T0~T3)
- **زر ?**: اطلب من الذكاء الاصطناعي شرح هذه المهارة
- **زر التبديل**: تمكين/تعطيل المهارات المخصصة

## البحث والإدارة الجماعية

- **البحث**: التصفية حسب اسم المهارة أو الوصف
- **تمكين الكل**: تفعيل جميع المهارات المخصصة
- **تعطيل الكل**: إلغاء تفعيل جميع المهارات المخصصة
- يتم عرض العدد النشط/الإجمالي (على سبيل المثال، 45/50)

## إدارة المهارات عبر الذكاء الاصطناعي

يمكنك أيضًا أن تطلب من AI 아바타 إدارة المهارات في الدردشة:

- "أظهر لي قائمة المهارات المتاحة"
- "هل توجد مهارة متعلقة بالطقس؟"
- "عطّل مهارة healthcheck"
- "ابحث عن المهارات المتعلقة بالبرمجة"

سيستخدم AI 아바타 أداة `skill_skill_manager` تلقائيًا.
