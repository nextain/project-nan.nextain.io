उपलब्ध कौशल (टूल) ब्राउज़ करें और प्रबंधित करें।

![Skills tab](skills-tab.png)

## कौशल के प्रकार

### अंतर्निहित कौशल
ऐप में एम्बेडेड — अक्षम नहीं किए जा सकते हैं:

| कौशल | कार्य | सुरक्षा स्तर |
|-------|----------|---------------|
| `skill_time` | वर्तमान तिथि/समय जांचें | T0 |
| `skill_memo` | मेमो सहेजें/पुनर्प्राप्त करें | T0 |
| `skill_system_status` | सिस्टम की स्थिति जांचें | T0 |
| `skill_weather` | मौसम जांचें | T0 |
| `skill_notify_slack` | Slack वेबहुक के माध्यम से सूचनाएं भेजें | T1 |
| `skill_notify_discord` | Discord वेबहुक के माध्यम से सूचनाएं भेजें | T1 |
| `skill_skill_manager` | कौशल प्रबंधित करें: खोजें, सक्षम करें, अक्षम करें | T0 |

### कस्टम कौशल
Gateway के माध्यम से जोड़े गए — चालू/बंद किए जा सकते हैं:
- फ़ाइल पढ़ें/लिखें, कमांड निष्पादन, वेब खोज, आदि।
- Gateway या Command प्रकार

## कौशल स्रोत (वे कहाँ से आते हैं?)

- **अंतर्निहित कौशल**: ऐप के साथ बंडल किए गए
- **कस्टम कौशल**: स्थानीय कौशल मैनिफेस्ट से लोड किए जाते हैं (उदाहरण के लिए, `~/.naia/skills/.../skill.json`)
- इसके `source` बैज की जांच के लिए एक कौशल कार्ड का विस्तार करें

## कस्टम कौशल कैसे जोड़ें

Naia OS OpenClaw इकोसिस्टम के साथ 100% संगत है। कौशल जोड़ने के तीन तरीके हैं:

### 1. AI को इसे बनाने के लिए कहें (सबसे आसान)
चैट में बताएं कि आप क्या चाहते हैं, और AI 아바타 आपके लिए कोड लिखेगा और कौशल बनाएगा।
> "Build a skill that fetches the current exchange rate and save it to `~/.naia/skills/exchange/skill.json`."

### 2. Clawhub से इंस्टॉल करें (OpenClaw तरीका)
आप **[Clawhub.ai](https://clawhub.ai)** से सीधे प्लगइन्स इंस्टॉल करने के लिए टर्मिनल (`execute_command`) टूल का उपयोग कर सकते हैं, जो 5,700 से अधिक कौशल वाला आधिकारिक OpenClaw कौशल रजिस्ट्री है।

> "Run `openclaw plugins install @openclaw/plugin-github` in the terminal to install the Github plugin."

⚠️ **सुरक्षा चेतावनी:** Clawhub या अन्य ऑनलाइन स्रोतों से डाउनलोड किए गए कौशल तीसरे पक्षों द्वारा लिखे गए हैं। इंस्टॉल करने से पहले, AI से हमेशा पूछें कि **"आगे बढ़ने से पहले इस कौशल के कोड को किसी भी सुरक्षा जोखिम (जैसे फ़ाइलें हटाना या व्यक्तिगत जानकारी चुराना) के लिए समीक्षा करें।"**

### 3. मैन्युअल जोड़ना
1. `~/.naia/skills/<skill-name>/skill.json` पर एक कौशल मैनिफेस्ट बनाएं
2. उस कौशल के लिए आवश्यक किसी भी स्क्रिप्ट/एक्जीक्यूटेबल को उसी फ़ोल्डर में रखें
3. स्किल्स टैब खोलें और जांचें कि क्या नया कौशल दिखाई देता है
4. टॉगल का उपयोग करके इसे सक्षम करें
5. चैट से एक अनुरोध के साथ इसका परीक्षण करें जो कौशल को ट्रिगर करना चाहिए

यदि यह दिखाई नहीं देता है, तो ऐप को पुनरारंभ करें और फिर से जांचें।

## बॉटमाडांग समुदाय एकीकरण

Naia OS में **बॉटमाडांग** के लिए समर्पित एक अंतर्निहित कौशल (`skill_botmadang`) शामिल है, जो एक कोरियाई AI एजेंट समुदाय है।

आप AI 아바타 को चैट के माध्यम से बॉटमाडांग पर अपनी गतिविधियाँ शुरू करने का निर्देश दे सकते हैं:
> "Register as a new agent on Botmadang. Set your name to 'naia Agent'."

एक बार पंजीकृत होने और एक API Key दिए जाने के बाद, AI 아바타 स्वायत्त रूप से लेख पोस्ट कर सकता है या अन्य एजेंटों के थ्रेड्स पर टिप्पणी कर सकता है।

## अधिसूचना कौशल (Slack / Discord / Google Chat)

`skill_notify_slack` और `skill_notify_discord` अंतर्निहित अधिसूचना कौशल हैं जो वेबहुक के माध्यम से संदेश भेजते हैं।

### वेबहुक सेटअप

अधिसूचना कौशल का उपयोग करने के लिए, आपको एक वेबहुक URL कॉन्फ़िगर करने की आवश्यकता है। दो तरीके हैं:

**तरीका 1: पर्यावरण चर (अनुशंसित)**

```bash
# Add to ~/.bashrc or ~/.zshrc
export SLACK_WEBHOOK_URL="https://hooks.slack.com/services/T.../B.../xxx"
export DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/123/abc"
```

**तरीका 2: config.json**

```json
// ~/.naia/config.json
{
  "notifications": {
    "slack": {
      "webhookUrl": "https://hooks.slack.com/services/T.../B.../xxx"
    },
    "discord": {
      "webhookUrl": "https://discord.com/api/webhooks/123/abc"
    }
  }
}
```

> पर्यावरण चर config.json पर प्राथमिकता लेते हैं।

### उपयोग के उदाहरण

बस चैट में AI 아바타 से पूछें:

- "Slack को 'deploy complete' अधिसूचना भेजें"
- "Discord पर सर्वर स्थिति रिपोर्ट पोस्ट करें"
- "बिल्ड परिणामों के साथ #ops चैनल को सूचित करें"

AI 아바타 स्वचालित रूप से `skill_notify_slack` या `skill_notify_discord` को कॉल करेगा।

यदि कोई वेबहुक कॉन्फ़िगर नहीं किया गया है, तो सेटअप चरणों की व्याख्या करने वाला एक संदेश दिखाया जाएगा।

### OpenClaw Gateway एकीकरण (उन्नत)

जब एक OpenClaw Gateway कनेक्ट होता है, तो अधिसूचना कौशल पहले Gateway के `skills.invoke` RPC का उपयोग करने का प्रयास करेंगे। यदि Gateway रिले विफल रहता है, तो कौशल सीधे वेबहुक डिलीवरी पर वापस आ जाता है।

Gateway चैनल एकीकरण अधिक समृद्ध सुविधाएँ (संदेश स्वरूपण, थ्रेड्स, उल्लेख, आदि) प्रदान करता है।

## उन्नत परिदृश्य: OpenClaw + cron स्वचालन

टीम/व्यक्तिगत स्वचालन सेटअप में, आप OpenClaw में कौशल पंजीकृत कर सकते हैं और उन्हें cron के साथ एक शेड्यूल पर ट्रिगर कर सकते हैं।

उदाहरण परिदृश्य:
- दैनिक 09:00: कल के कार्य लॉग का सारांश उत्पन्न करें
- प्रति घंटा: एक लक्षित फ़ोल्डर को स्कैन करें और विसंगतियों पर सूचित करें
- आधी रात: एक दैनिक रिपोर्ट उत्पन्न करें और अपलोड करें

अनुशंसित प्रवाह:
1. कस्टम कौशल को पंजीकृत करें और पहले इसे स्थानीय रूप से मान्य करें
2. अलर्ट चैनलों को जोड़ने के लिए अधिसूचना कौशल वेबहुक कॉन्फ़िगर करें
3. अपनी OpenClaw कार्य परिभाषा में एक कौशल आह्वान चरण जोड़ें
4. आवर्ती ट्रिगर के रूप में एक cron शेड्यूल संलग्न करें
5. विफलताओं के लिए पुनः प्रयास/अधिसूचना नीतियां जोड़ें

> **रोडमैप**: cron शेड्यूलिंग UI, टेलीग्राम समर्थन, और मल्टी-चैनल रूटिंग (एक साथ कई चैनलों पर एक संदेश भेजना) भविष्य के अपडेट में उपलब्ध होंगे।

## कौशल कार्ड

प्रत्येक कौशल को एक कार्ड के रूप में प्रदर्शित किया जाता है:

![Skill card detail](skills-card.png)

- **नाम**: कौशल का नाम (जैसे, `skill_read_file`)
- **विवरण**: एक-लाइन का सारांश (छोटा किया जा सकता है)
- **क्लिक करें**: पूर्ण विवरण का विस्तार करने के लिए कार्ड पर क्लिक करें
- **बैज**: प्रकार (अंतर्निहित/gateway/कमांड), सुरक्षा स्तर (T0~T3)
- **? बटन**: इस कौशल को समझाने के लिए AI से पूछें
- **टॉगल**: कस्टम कौशल सक्षम/अक्षम करें

## खोज और थोक प्रबंधन

- **खोज**: कौशल नाम या विवरण के अनुसार फ़िल्टर करें
- **सभी सक्षम करें**: सभी कस्टम कौशल सक्रिय करें
- **सभी अक्षम करें**: सभी कस्टम कौशल निष्क्रिय करें
- सक्रिय/कुल गणना प्रदर्शित (उदाहरण के लिए, 45/50)

## AI के माध्यम से कौशल प्रबंधित करें

आप चैट में AI 아바타 को कौशल प्रबंधित करने के लिए भी कह सकते हैं:

- "मुझे उपलब्ध कौशल की सूची दिखाएं"
- "क्या मौसम से संबंधित कोई कौशल है?"
- "हेल्थचेक कौशल को अक्षम करें"
- "कोडिंग से संबंधित कौशल खोजें"

AI 아바타 स्वचालित रूप से `skill_skill_manager` टूल का उपयोग करेगा।

## सुरक्षा स्तर

| स्तर | विवरण | अनुमोदन |
|------|------------|----------|
| T0 | केवल-पढ़ने योग्य, कोई दुष्प्रभाव नहीं | स्वतः-अनुमोदित |
| T1 | केवल अधिसूचना | सूचना दिखाई गई |
| T2 | सावधानी आवश्यक | उपयोगकर्ता अनुमोदन की आवश्यकता है |
| T3 | खतरनाक ऑपरेशन | उपयोगकर्ता अनुमोदन की आवश्यकता है |
