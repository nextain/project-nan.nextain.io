## Типы навыков

### Встроенные навыки
Встроены в приложение — не могут быть отключены:

| Навык | Функция | Уровень безопасности |
|-------|----------|---------------|
| `skill_time` | Проверить текущую дату/время | T0 |
| `skill_memo` | Сохранять/получать заметки | T0 |
| `skill_system_status` | Проверить статус системы | T0 |
| `skill_weather` | Проверить погоду | T0 |
| `skill_notify_slack` | Отправлять уведомления через Slack webhook | T1 |
| `skill_notify_discord` | Отправлять уведомления через Discord webhook | T1 |
| `skill_skill_manager` | Управлять навыками: поиск, включение, отключение | T0 |

### Пользовательские навыки
Добавляются через Gateway — можно включать/отключать:
- Чтение/запись файлов, выполнение команд, веб-поиск и т.д.
- Тип Gateway или Command

## Источники навыков (Откуда они берутся?)

- **Встроенные навыки**: поставляются в комплекте с приложением
- **Пользовательские навыки**: загружаются из локальных манифестов навыков (например, `~/.naia/skills/.../skill.json`)
- Разверните карточку навыка, чтобы проверить его значок `source`

## Как добавить пользовательский навык

Naia OS на 100% совместима с экосистемой OpenClaw. Есть три способа добавить навыки:

### 1. Попросить ИИ создать его (самый простой)
Объясните, что вы хотите, в чате, и AI 아바타 напишет код и создаст навык для вас.
> "Создай навык, который получает текущий обменный курс и сохраняет его в `~/.naia/skills/exchange/skill.json`."

### 2. Установка из Clawhub (способ OpenClaw)
Вы можете использовать инструмент Terminal (`execute_command`) для установки плагинов непосредственно с **[Clawhub.ai](https://clawhub.ai)**, официального реестра навыков OpenClaw, содержащего более 5700 навыков.

> "Запусти `openclaw plugins install @openclaw/plugin-github` в терминале, чтобы установить плагин Github."

⚠️ **Предупреждение о безопасности:** Навыки, загруженные из Clawhub или других онлайн-источников, написаны третьими лицами. Перед установкой всегда просите ИИ **"Проверить код этого навыка на наличие каких-либо рисков безопасности (таких как удаление файлов или кража личной информации), прежде чем продолжить."**

### 3. Ручное добавление
1. Создайте манифест навыка по адресу `~/.naia/skills/<skill-name>/skill.json`
2. Поместите любой необходимый скрипт/исполняемый файл для этого навыка в ту же папку
3. Откройте вкладку Skills и проверьте, появился ли новый навык
4. Включите его с помощью переключателя
5. Протестируйте его в чате с запросом, который должен запустить навык

Если он не появляется, перезапустите приложение и проверьте снова.

## Интеграция с сообществом Botmadang

Naia OS включает встроенный навык (`skill_botmadang`), предназначенный для **Botmadang**, корейского сообщества ИИ-агентов.

Вы можете поручить AI 아바타 начать свою деятельность на Botmadang через чат:
> "Зарегистрируйся как новый агент на Botmadang. Установи свое имя на 'naia Agent'."

После регистрации и получения API Key, AI 아바타 может автономно публиковать статьи или комментировать ветки других агентов.

## Навыки уведомлений (Slack / Discord / Google Chat)

`skill_notify_slack` и `skill_notify_discord` — это встроенные навыки уведомлений, которые отправляют сообщения через вебхуки.

### Настройка вебхуков

Чтобы использовать навыки уведомлений, вам необходимо настроить URL-адрес вебхука. Есть два метода:

**Метод 1: Переменные окружения (рекомендуется)**

```bash
# Добавьте в ~/.bashrc или ~/.zshrc
export SLACK_WEBHOOK_URL="https://hooks.slack.com/services/T.../B.../xxx"
export DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/123/abc"
```

**Метод 2: config.json**

```json
// ~/.naia/config.json
{
  "notifications": {
    "slack": {
      "webhookUrl": "https://hooks.slack.com/services/T.../B.../xxx"
    },
    "discord": {
      "webhookUrl": "https://discord.com/api/webhooks/123/abc"
    }
  }
}
```

> Переменные окружения имеют приоритет над config.json.

### Примеры использования

Просто спросите AI 아바타 в чате:

- "Отправь уведомление 'развертывание завершено' в Slack"
- "Опубликуй отчет о состоянии сервера в Discord"
- "Уведоми канал #ops о результатах сборки"

AI 아바타 автоматически вызовет `skill_notify_slack` или `skill_notify_discord`.

Если вебхук не настроен, будет показано сообщение, объясняющее шаги настройки.

### Интеграция с OpenClaw Gateway (расширенная)

Когда подключен OpenClaw Gateway, навыки уведомлений сначала попытаются использовать RPC `skills.invoke` Gateway. Если передача через Gateway не удается, навык возвращается к прямой доставке через вебхук.

Интеграция с каналами Gateway предоставляет более богатые функции (форматирование сообщений, потоки, упоминания и т.д.).

## Продвинутый сценарий: OpenClaw + автоматизация cron

В командных/личных сценариях автоматизации вы можете регистрировать навыки в OpenClaw и запускать их по расписанию с помощью cron.

Примеры сценариев:
- Ежедневно в 09:00: генерировать сводку вчерашних журналов работы
- Ежечасно: сканировать целевую папку и уведомлять об аномалиях
- В полночь: генерировать и загружать ежедневный отчет

Рекомендуемый рабочий процесс:
1. Зарегистрируйте пользовательский навык и сначала проверьте его локально
2. Настройте вебхуки навыков уведомлений для подключения каналов оповещений
3. Добавьте шаг вызова навыка в определение задачи OpenClaw
4. Прикрепите расписание cron в качестве повторяющегося триггера
5. Добавьте политики повторных попыток/уведомлений о сбоях

> **План развития**: пользовательский интерфейс для планирования cron, поддержка Telegram и многоканальная маршрутизация (отправка одного сообщения в несколько каналов одновременно) будут доступны в будущих обновлениях.

## Карточки навыков

Каждый навык отображается в виде карточки:

![Skill card detail](skills-card.png)

- **Имя**: Название навыка (например, `skill_read_file`)
- **Описание**: Однострочное резюме (может быть усечено)
- **Нажмите**: Нажмите на карточку, чтобы развернуть полное описание
- **Значки**: Тип (built-in/gateway/command), уровень безопасности (T0~T3)
- **Кнопка ?**: Попросить ИИ объяснить этот навык
- **Переключатель**: Включить/отключить пользовательские навыки

## Поиск и массовое управление

- **Поиск**: Фильтрация по названию или описанию навыка
- **Включить все**: Активировать все пользовательские навыки
- **Отключить все**: Деактивировать все пользовательские навыки
- Отображается активное/общее количество (например, 45/50)

## Управление навыками через ИИ

Вы также можете попросить AI 아바타 управлять навыками в чате:

- "Покажи список доступных навыков"
- "Есть ли навык, связанный с погодой?"
- "Отключи навык проверки работоспособности"
- "Найди навыки, связанные с кодированием"

AI 아바타 автоматически использует инструмент `skill_skill_manager`.

## Уровни безопасности

| Уровень | Описание | Утверждение |
|------|------------|----------|
| T0 | Только чтение, без побочных эффектов | Автоматическое утверждение |
| T1 | Только уведомление | Показывается уведомление |
| T2 | Требуется осторожность | Требуется подтверждение пользователя |
| T3 | Опасная операция | Требуется подтверждение пользователя |
