# 개발 사이클 — lab.cafelua.com

> 이 문서는 `.agents/workflows/development-cycle.yaml`의 한국어 미러입니다.
> **코딩 작업 전 반드시 읽어주세요.**

## 사이클

```
PLAN → CHECK → BUILD → VERIFY → CLEAN → COMMIT
```

### 1. PLAN (계획)
- 사용자와 요구사항 명확화
- 영향받는 파일 식별
- 기존 코드에서 중복 확인
- **산출물**: 명확한 범위 + 파일 목록

### 2. CHECK (확인)
- 코드베이스에서 관련 패턴 검색
- 수정 전 관련 파일 읽기
- 충돌하는 변경 없는지 확인
- **산출물**: 현재 상태 이해

### 3. BUILD (구현)
**TDD 필수!**
- **RED**: 기능/수정에 대한 실패하는 테스트 작성
- **GREEN**: 통과시키는 최소 코드 작성
- **REFACTOR**: 테스트 통과 유지하면서 정리

규칙:
- 테스트 없이 코딩 금지
- 통합 테스트 우선 (유닛 테스트보다)
- console.log 금지

### 4. VERIFY (검증)
- 테스트 실행 (`npm test`)
- 개발 서버 실행 (`npm run dev`) + 수동 확인
- UI 변경이면 브라우저에서 직접 확인

**중요: 타입 체크만으로는 검증이 아닙니다. 반드시 앱을 실행해야 합니다.**

### 5. CLEAN (정리)
- 미사용 import/파일 제거
- 중복 코드 확인
- 모든 변경사항 셀프 리뷰

### 6. COMMIT (커밋)
- 특정 파일만 스테이지 (`git add -A` 금지)
- 커밋: `<type>(<scope>): <description>`
- 중요한 작업이면 work-logs 업데이트
